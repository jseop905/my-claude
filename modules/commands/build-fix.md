# /build-fix

> 빌드 실패 시 에러를 하나씩 분석하고 수정. 최소 변경으로 빌드를 통과시킴.

## 사용법

```
/build-fix
```

## 실행 절차

### Step 1: 빌드 실행 & 에러 수집

1. 프로젝트 빌드 명령어 실행
2. 에러 출력 전체 캡처
3. 파일별, 심각도별 그룹핑

### Step 2: 에러별 순차 수정

각 에러에 대해:

1. **컨텍스트 확인**: 에러 발생 파일의 관련 코드 읽기
2. **원인 분석**: 에러 메시지 해석, 근본 원인 파악
3. **수정 제안**: 최소 변경 방안 결정
4. **적용**: 코드 수정
5. **재빌드**: 빌드 재실행으로 수정 확인

### Step 3: 결과 보고

```
## /build-fix 결과

- 초기 에러: 12개
- 수정 완료: 11개
- 미수정 (수동 필요): 1개
  - src/auth.ts:45 — 비즈니스 로직 변경 필요

### 다음 단계
- /verify-loop: 전체 검증 실행
- /commit: 수정 사항 커밋
```

## 중단 조건

- 수정 후 새로운 에러가 발생한 경우 → 롤백 후 보고
- 동일 에러가 3회 반복되는 경우 → 수동 수정 필요로 분류
- 사용자가 중단 요청한 경우

## 핵심 규칙

- 최소 변경만 적용 — "ついでに" 개선 금지
- 에러 하나씩 순차 수정, 수정 후 반드시 재빌드 검증
- 비즈니스 로직 변경이 필요한 에러는 수동 수정으로 분류
- 테스트 assertion 변경으로 테스트를 통과시키지 마라 (구현을 수정)
