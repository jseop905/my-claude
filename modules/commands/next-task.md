# /next-task

> 작업 계획에서 진행 상황을 파악하고, 다음에 할 태스크를 추천.

## 사용법

```
/next-task
/next-task <task-number>
```

- 인자 없으면: 자동으로 다음 태스크 추천
- 인자 있으면: 해당 태스크 상세 분석

## 실행 절차

### Step 1: 프로젝트 상태 수집

1. `git status`로 현재 작업 상태 확인
2. `prompt_plan.md` 읽기 (작업 계획)
3. 관련 문서 확인 (CLAUDE.md, spec.md 등)

### Step 2: 진행률 계산

```
## 진행 현황

전체: 12개 태스크
- ✅ 완료: 5개 (42%)
- 🔄 진행중: 1개
- ⬜ 미시작: 6개

진행률: ████████░░░░░░░░░░░░ 42%
```

### Step 3: 다음 태스크 추천

선택 기준 (우선순위 순):
1. 사용자가 지정한 태스크
2. 의존성이 해결된 태스크 (선행 태스크 완료)
3. 순서상 다음 태스크

### Step 4: 태스크 상세 분석

```
## 추천: Task 6 - 사용자 인증 API

- 복잡도: Medium (3-5 파일, ~200줄)
- 관련 파일: src/auth/, src/middleware/
- 의존성: Task 3 (완료), Task 5 (완료)
- 추천 접근: /auto feature "사용자 인증 API 구현"
```

## 복잡도 기준

| 복잡도 | 파일 수 | 예상 줄 수 | 추천 워크플로우 |
|--------|---------|-----------|---------------|
| Small | 1-2 | ~50 | 직접 구현 → /commit |
| Medium | 3-5 | ~200 | /auto feature |
| Large | 6+ | ~500+ | /plan 먼저 → 분할 |
