# /plan

planner 에이전트를 호출하여 구현 계획을 작성한다.

## 실행 절차

1. **요구사항 분석** — 사용자 요청을 명확히 재정리
2. **코드베이스 조사** — Read/Grep/Glob으로 관련 코드 파악 (사용자에게 묻지 않음)
3. **질문** — 우선순위/범위/리스크만 질문 (한 번에 하나씩)
4. **계획 생성** — 3-6단계 구현 계획 작성
5. **확인 대기** — 사용자가 승인할 때까지 대기

## 핵심 규칙

- **사용자가 "yes" 할 때까지 코드 작성 금지**
- 코드베이스 질문은 직접 조사 (사용자에게 묻지 않음)
- 3-6단계가 적정. 30개 마이크로스텝이나 2개 모호한 지시 금지

## 확인 후 처리

사용자가 계획을 확인하면:
1. `prompt_plan.md`에 계획 저장
2. 기존 파일이 있으면 "## 이전 계획" 섹션으로 아카이브 후 덮어쓰기
3. 안내: "계획이 prompt_plan.md에 저장되었습니다."

## 수정 요청

사용자가 수정을 원하면:
- "modify: [변경 내용]"
- "다른 접근: [대안]"
- "Phase 2를 먼저 진행"
